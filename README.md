# Тренер у дома

Продакшен-лендинг персонального тренера с интерактивным тестом, выбором времени тренировки и отправкой заявки в Telegram.

- Сайт: https://trenerprimorskiy.ru/
- Репозиторий: https://github.com/thermojam/athome

## О проекте

Сайт помогает пользователю познакомиться с тренером, понять формат занятий, увидеть расположение зала и оставить заявку без регистрации и сложных форм.

Основная задача проекта — провести посетителя от первого знакомства до понятного действия: пройти тест, выбрать удобное время и написать тренеру в Telegram.

## Основные возможности

- адаптивный одностраничный интерфейс;
- информация о тренере и формате занятий;
- блок с расположением зала;
- описание проблем и ожидаемых изменений;
- интерактивный тест из нескольких вопросов;
- определение одного из трёх профилей пользователя;
- персональный результат и предложение после теста;
- блок ответов на возражения;
- выбор доступного времени тренировки;
- создание Telegram-сообщения с выбранным временем;
- фиксированная кнопка записи;
- цели Яндекс.Метрики для важных действий;
- загрузка аналитики только после согласия пользователя;
- баннер и настройки cookies;
- страницы политики конфиденциальности и cookies;
- SEO-метаданные, canonical, robots.txt и sitemap.xml;
- unit-, component- и e2e-тесты.

## Архитектура

Большая часть страницы собрана на Server Components. Клиентские компоненты используются только там, где нужна интерактивность:

- тест;
- выбор времени;
- аккордеон с возражениями;
- аналитика;
- управление согласием на cookies.

Контент, логика теста, данные свободных слотов и работа с Telegram вынесены из UI-компонентов.

```text
athome/
├── app/                    # страницы, layout и глобальные стили
├── components/
│   ├── analytics/          # аналитика и согласие пользователя
│   ├── legal/              # элементы для cookies и политики
│   ├── quiz/               # интерактивный тест
│   ├── sections/           # секции лендинга
│   └── ui/                 # общие UI-компоненты
├── lib/                    # данные, типы и бизнес-логика
├── public/                 # изображения, логотип и шрифты
└── tests/                  # unit-, component- и e2e-тесты
```

## Локальный запуск

### Требования

- Node.js;
- npm.

### 1. Клонирование и установка

```bash
git clone https://github.com/thermojam/athome.git
cd athome
npm install
```

### 2. Переменные окружения

Создайте файл `.env.local`:

```env
NEXT_PUBLIC_TG_USERNAME=your_telegram_username
NEXT_PUBLIC_YM_ID=your_yandex_metrika_id
```

Имя Telegram указывается без символа `@`.

Для локальной разработки счётчик Метрики можно не указывать. В этом случае сайт продолжит работать без аналитики.

### 3. Запуск

```bash
npm run dev
```

Откройте:

```text
http://localhost:3000
```

## Проверка проекта

```bash
npm run lint       # ESLint
npm run typecheck  # проверка TypeScript
npm test           # тесты Vitest
npm run test:e2e   # e2e-тесты Playwright
npm run build      # production-сборка
```

Для первого запуска e2e-тестов может понадобиться установка браузеров Playwright:

```bash
npx playwright install
```

## Основной пользовательский сценарий

1. Пользователь знакомится с предложением и тренером.
2. Смотрит расположение зала и формат тренировок.
3. Проходит короткий тест.
4. Получает персональный результат.
5. Выбирает подходящее время.
6. Переходит в Telegram с готовым сообщением.
